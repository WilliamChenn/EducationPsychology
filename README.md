# Motivation, Reinforcement, and Learning Environments in Math Tutoring

An interactive educational psychology module that connects course theory to a virtual math
tutoring service-learning placement.

## 1. Course

EDUC 240 / Educational Psychology

## 2. Project Description

This is a static, interactive website built for the final project (Option 1: Website / Multimedia /
EdTech Project). It is a self-contained educational module — built with plain HTML, CSS, and vanilla
JavaScript — that teaches key educational psychology concepts and shows how they apply to a real
one-on-one math tutoring experience. The site includes explanatory chapters, a concept map, a
service-learning case study with a timeline, and several interactive practice activities (scenario
responses, a feedback sorter, a session builder, and an eight-question quiz).

The site requires no build tools, frameworks, package managers, or backend. It works by opening
`index.html` directly in any modern browser.

## 3. Concepts Covered

- Expectancy theory
- Attribution theory
- Intrinsic and extrinsic motivation
- Reinforcement and shaping
- Modeling / observational learning
- Effective learning environments and engaged time

Supporting ideas also addressed: self-efficacy, specific feedback, routines and expectations,
structure with flexibility, and the least intrusive intervention.

## 4. Service-Learning Connection

The project draws on a virtual math tutoring placement with a 10th grade student. Sessions
covered homework review, geometry (angles and parallel lines, transformations, congruence and
similarity, the triangle inequality), right-triangle trigonometry, geometry vocabulary such as
*theta* and *apothem*, and EOG/EOC-style test preparation. To protect privacy, the student is not
named and identifying details are omitted.

## 5. Files

- `index.html` — all page content and structure (14 sections, single page)
- `styles.css` — design system, layout, responsive and print styles
- `script.js` — navigation, tabs, and the four interactive activities
- `README.md` — this file

## 6. How to Run Locally

Open `index.html` in any modern browser (double-click the file, or drag it into a browser window).
No server or install step is required. The page uses the Instrument Sans, Instrument Serif, and Inter
typefaces loaded from Google Fonts for a clean academic look; if the device is offline, the layout
falls back gracefully to system fonts and remains fully functional.

## 7. GitHub Pages Deployment

1. Push the project to a GitHub repository.
2. In the repository, go to **Settings**.
3. In the left sidebar, go to **Pages**.
4. Under **Build and deployment → Source**, select **Deploy from a branch**.
5. Choose the **main** branch and the **/ (root)** folder, then click **Save**.
6. Wait about a minute, then open the published URL shown on the Pages settings screen.

Because every file is static and self-contained (no CDN links, APIs, or external assets), the site
runs identically locally and on GitHub Pages.
